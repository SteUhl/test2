## Our Scenario and pipeline

In this lesson we will combine teaching the basic elements of Quarto with the process of writing and publishing a scientific articles.
To do so, we will create a fictional paper, focussed on the distribiution of different medias of art in the Museum of Modern Arts (MOMA).

We will beginn by after we have finished our research and are about to start actually writing our paper. To do so we need to we need to go through several steps, all of which we will need to complete before we can move on.

We already started by installing Quarto and our preferred coding environment on our computer. For this we chose Visual Studio Code.
Now we need to actually start writing the paper in Quarto. 

To do so we will first create our project folder and document. This willbe done through the "getting Started" episode.

Then we want to start writing our text. We quickly notice that we need to learn the basics of the "Markdown" language, which is used in Quarto to create text and syntax. This will we done throughout our "Markdown Basics" episode.

After we used our newly gained Markdown knowledge to structure and write down the text of our paper, we need to enrich our paper with the necessary metadata, which is then used to create a header and front page. In order to do this we will need another language used in Quarto: YAML. This will allbe shown in our "An introduction to YAML headers" episode.

We are halfway done with our paper. Now that we have a structured text, enriched with metadata, we want to spruce ot up with a few relevant and visually pleasing ilustrations and graphs. This can be done though the in Quarto integrated visual coding languages, such as Mermaid or Graphviz. To do so, we walk through the staps of the next episode: "Creating and implementing code for visualisations".

Now we are almost done. There are only a few more steps to be done. First we want to decide if we want our paper to be published as a pdf or as an HTML document. This and the unique option we have for these formats will be discussed in the short "Quarto for PDF and HTML" episode.

For our paper to be considered as academically we naturally need to provide an abundent amount of sources in form of footnotes, literature and and solid bibliography. Thankfully Quarto offers us a nice and easy to understand integrated bibliography feature, which we will ise to enrich our paper with sources. This will be shown and done thorughout our episode "Using the Bibliography".

Finally we are finished with our paper! But now the option to create a complimentary website, on which we present our findings, sound very enticing. So we will do so with the options Quarto offers us. This will be done in our last episode "Creating a simple website with Quarto" 


::: caution

Of course all of theseoptions are open to you at all pints. When using Quarto to publish you can freely combine all these functions and steps to create your own workflow!

:::
